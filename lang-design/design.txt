# 独自言語仕様書 (Language Specification)

## 1. 言語のコア哲学 (Philosophy)
* **ノーシフト美学**: `()` や `=` などのShiftキーを要する記号を極力排除し、タイピングの負荷と視覚的なノイズを減らす。
* **すべてがメッセージ送信**: 制御構文（if, loop等）や演算子（+, -）もすべてオブジェクトへのメッセージ送信（`AST_SEND`）として統一される純粋オブジェクト指向。
* **状態（State）と振る舞い（Behavior）の分離**:
    * `.cd` ファイル（クラス定義）: 状態の定義。再代入（`is`）が許可される。
    * (拡張子未定) ファイル（動作記述）: 振る舞いの定義。**再代入禁止**。変数は宣言と同時に式を紐付け(コンストラクタでの初期化)、以降は不変（副作用の排除）内部状態の更新のみでプログラムは記述される。
* **静的型付け**: コンパイル時（AST構築時）に厳密な型チェックを行い、安全性を担保する。

---

## 2. 基本構文 (Syntax)

### 文の終端
すべての文（ステートメント）はピリオド `.` で終わる。

### 変数の宣言と紐付け
型を明示し、コロン `:` を用いて宣言する。
'''text
/* 宣言と初期化 (クラス定義) */
// フィールド変数宣言
name: String.
name: String is "Taro." // デフォルト指定
arr: List Integer. // 配列宣言 (型(List)の後にジェネリクス)
arr: List Integer are 1, 2, 3. // デフォルト指定
// ブロック内の表記
name: String new "Taro". // newはコンストラクタ
name is "Jiro". // 宣言後は型指定はいらないかつクラス定義ファイルなら再代入も可能
/* 宣言と初期化 (動作記述) */
name: String new "Taro". // 動作記述では宣言時にコンストラクタの使用を厳格化
// デフォルトコンストラクタでnewに引数がない場合はデフォルトの値でインスタンスが生成される
'''

### 再代入 (クラス定義のメソッドのみ許可)
'''text
// is キーワードを使用
age is 26 .
'''

### メッセージ送信
レシーバに対してメッセージ名と引数をスペース区切りで並べる。
'''text
// メッセージ送信 (レシーバ メッセージ名 引数 .)
calc add 10 20 .
print result. // レシーバがないメッセージ送信はVMがレシーバとなる(Ruby方式)

// 演算子もメッセージ送信として扱われる
x + y . 
now > 0 .
'''

## 3. 数値と型システム (Type System & Literals)

VMの空気を読まない（自動昇格しない）厳格な数値管理。プログラマが明示的に使い分ける。

* **Integer (整数)**
    * 32bitで爆速に計算される。
    * `1 / 2` （スペースあり）は Integer クラスの割り算となり、結果は `0`（切り捨て）。
* **Fraction (分数 / 多倍長10進数)**
    * 自作の `Number` / `List` ライブラリを裏で駆動する無限精度の数値型。
    * **Lexerハック**: `1/2` （スペースなし）と書くと、Lex層で `FRACTION` トークンとしてパースされ、AST構築時に最初から Fraction オブジェクトとして定数畳み込みされる。

'''text
// Integerの計算 (動作記述)
a: Integer new 10 .
b: Integer new 3 .
ans: Integer new a / b . // 3

// Fractionの計算
x: Fraction new 1/2 .
y: Fraction new 1/4 .
ans: Fraction new x + y . // 内部で Fraction(3/4) が生成される
'''

## 4. 制御フローとブロック (Control Flow & Blocks)

ブロック `[ ... ]` を用いて、クロージャやメソッドのスコープを表現する。

### 透過ブロックと `ret` の壁
ブロック自体は「透過的」であり、スコープやジャンプ命令（`return`, `break`）は外側の壁に向かって突き抜ける。ただし、`ret` 宣言があるブロックは「壁」となる。

'''text
// 引数と戻り値の型を持つブロック（壁になる）
[ arg a: Integer, b: Integer . ret Integer . 
    return a + b . 
] 
'''

### ジャンプ命令 (専用ASTノード)
これらはメッセージ送信ではなく、実行コンテキストに対する直接的な制御命令。
* `return <expr> .` : 最も近い `ret` の壁、またはメソッドの壁を抜けて値を返す。
* `break .` : 値を持たず、ループの壁をぶち破って反復を終了する（Stop）。
* `continue <expr> .` : 現在の周回を抜け、必要に応じて次の周・mapなどに値を渡す。

## 5. クラス定義 (.cd)

`class` 宣言を用い、`based` (継承) や `from` (Mix-in/委譲) をサポートする。
尚、メタクラスであるクラスクラスが存在する
'''text
class User based Person
    // フィールド宣言 (メタクラスより自動的に OP_GET_FIELD / OP_SET_FIELD のゲッターセッターが生成される)
    id: Integer .
    name: String .
    
    // フィールドの初期値設定
    status: String is "active" .
'''

## 6. コンパイラ & VM アーキテクチャ

### AST (抽象構文木)
驚異的なシンプルさを誇る、実質5+3種類のノード群。
1.  `AST_LITERAL` (定数・Lexerで評価済みのFraction含む)
2.  `AST_IDENTIFIER` (変数参照)
3.  `AST_ASSIGN` (変数代入/宣言 - 静的型チェック用)
4.  `AST_SEND` (メッセージ送信 - すべての主役)
5.  `AST_BLOCK` (ブロック - `has_ret_decl` フラグを持つ)
6.  (ジャンプ系: `AST_RETURN`, `AST_BREAK`, `AST_CONTINUE`)

### NaN Boxing (VMのメモリ管理)
スタックマシンの `Value` はすべて **8バイト (64bit)** に格納される。
* 浮動小数点: そのままの `double`
* Integer: 上位に `TAG_INT` (3ビット) + 下位に32bit整数
* Bool / Nil: `TAG_BOOL`, `TAG_NIL` などの即値
* オブジェクト: `TAG_OBJ` + 下位48bitにヒープ領域のポインタ (`Number*`, `String*`, `InstanceObj*` など)
